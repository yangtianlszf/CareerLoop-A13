# A13 官方资料说明

本目录存放第十七届中国大学生服务外包创新创业大赛 A13 赛题的官方资料，是整个项目的数据和需求来源。

## 目录内容

- `第十七届中国大学生服务外包创新创业大赛A13赛题.pdf`
- `A13-JD采样数据.xls`

## 文件用途

### 1. 赛题 PDF

用于确认项目边界、评分关注点和交付要求，重点包括：

- 至少构建 `10` 个就业岗位画像
- 建立岗位图谱、晋升路径和换岗路径
- 支持学生就业能力画像生成
- 输出完整职业生涯发展报告
- 报告需要可操作、可解释、可导出
- 关键技能匹配准确率、画像准确率和使用价值都需要达标

### 2. JD 样本 Excel

用于构建岗位库、岗位模板和证据基线，是系统推荐逻辑的数据基础。当前项目已经基于这份官方数据完成：

- JD 清洗与归一化
- 岗位名称标准化
- 岗位模板沉淀
- 原始样本检索
- 模板证据回看

## 赛题要求摘要

### 岗位画像

- 岗位画像数量不少于 `10`
- 覆盖专业技能、证书要求、创新能力、学习能力、抗压能力、沟通能力、实习能力等维度

### 岗位图谱与职业路径

- 需要体现岗位之间的关联关系
- 要体现垂直发展路径
- 换岗路径至少覆盖 `5` 个岗位
- 每个岗位至少给出 `2` 条换岗路径

### 学生画像

- 支持简历上传或手动录入
- 要能解析学生教育背景、技能、项目、实习、证书、竞赛等信息
- 需要给出完整度和竞争力判断

### 职业规划报告

- 包含人岗匹配分析
- 包含职业目标和职业路径建议
- 包含短期、中期行动计划
- 支持润色、编辑、导出

### 技术与评审关注点

- 至少接入 `1` 个大语言模型
- 推荐结果要有依据、可解释
- 界面与交互要能体现真实使用价值
- 不只是“会聊天”，而是要形成完整的职业规划服务闭环

## 本项目如何使用这些资料

- `赛题 PDF` 作为需求与验收基线
- `JD Excel` 作为岗位模板和证据库的数据源
- `a13_starter/tools/import_jd_xls.py` 用于导入与清洗官方数据
- `a13_starter/generated/job_profiles.jsonl` 保存清洗后的岗位记录
- `a13_starter/generated/role_profile_templates.json` 保存岗位模板库

## 提交与答辩注意事项

- 所有展示材料都应避免出现参赛院校名称等敏感信息
- PPT、视频、报告、网页演示内容要统一口径
- 所有“我们为什么这么推荐”的结论，尽量回到官方 JD 数据和模板证据上解释

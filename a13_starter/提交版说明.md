# 提交版说明

本文件用于整理最终提交包，目标是让提交版“能运行、能演示、能导出、能说明”。

## 1. 建议保留的内容

### 核心代码与页面

- `a13_starter/src/`
- `a13_starter/web/`
- `a13_starter/tools/`
- `a13_starter/samples/`
- `a13_starter/api_server.py`
- `a13_starter/requirements.txt`
- `a13_starter/Dockerfile`

### 数据与模板

- `A13_官方资料/`
- `a13_starter/generated/job_profiles.jsonl`
- `a13_starter/generated/role_profile_templates.json`
- `a13_starter/generated/role_library.json`
- `a13_starter/generated/dataset_summary.json`

### 文档

- `README.md`
- `a13_starter/README.md`
- `a13_starter/DEPLOY.md`
- `a13_starter/答辩讲解提纲.md`
- `a13_starter/答辩演示脚本.md`
- `a13_starter/desktop/README.md`
- `仓库根目录/.env.example`
- `仓库根目录/FINAL_DELIVERY_CHECKLIST.md`

### 可选保留

- `a13_starter/generated/analysis_history.db`
  如果希望提交版里自带历史记录和复核示例，可以保留

- `a13_starter/generated/career_plan_report.md`
  如果希望附上一份示例报告，可以保留

## 2. 建议删除的内容

- `__pycache__/`
- 多余的 `.pyc`
- 临时日志
- 本地测试残留文件
- 私有 Key、`.env.local`
- 与答辩无关的临时截图和个人开发文件

## 3. 提交包自检

提交前至少确认以下几点：

- 项目能从仓库根目录启动
- 样例简历能正常生成职业规划
- PDF 或 Word 导出至少有一种可正常使用
- 文档里的命令和路径与实际仓库结构一致
- 所有展示材料中不出现敏感信息与院校名称

## 4. 不要把什么当成正式文档

下面这些更偏运行产物，不建议当成项目主文档：

- `a13_starter/generated/*.md`
- 临时 benchmark 输出
- 历史记录数据库中的演示样本内容

## 5. 推荐交付口径

最终提交包应体现四件事：

1. 我们确实使用了官方资料
2. 我们做的是完整系统，而不是单点脚本
3. 我们的推荐结果有依据、有解释、有行动方案
4. 我们的作品可以被真实运行和演示

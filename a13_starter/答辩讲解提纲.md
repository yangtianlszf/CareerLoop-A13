# A13 答辩讲解提纲

## 1. 项目定位

- 这是一个面向大学生求职场景的职业规划智能体
- 系统核心目标是把“学生简历”与“真实岗位需求”连接起来
- 输出不仅是岗位推荐，还包括成长路径、转岗路径和阶段行动建议

## 2. 为什么不是普通推荐系统

- 我们不是只做关键词匹配
- 我们把官方 JD 数据沉淀成岗位模板库
- 再通过学生画像、岗位画像、匹配评分和解释性报告形成闭环

## 3. 数据来源

- 官方 A13 赛题提供的 JD 采样数据
- 共导入并整理了 9958 条真实 JD
- 在此基础上构建了岗位库和标准岗位模板

## 4. 系统链路

1. 输入学生简历文本或上传简历文件
2. 解析学生画像
3. 调用岗位模板进行匹配评分
4. 生成主推荐岗位、备选岗位和成长路径
5. 输出职业规划报告，并支持历史记录和导出

## 5. 项目亮点

- 支持规则解析和 LLM 解析双模式
- 支持 txt、md、docx 简历上传
- 支持历史记录持久化
- 支持 Markdown、HTML、Word 导出
- 页面内置职业图谱可视化，方便答辩展示

## 6. 解释性怎么体现

- 展示 Top 岗位分数不是终点
- 系统同时展示共享技能、关键差距、行动建议和风险提示
- 评委可以清楚看到“为什么推荐这个岗位”

## 7. 演示顺序建议

1. 先展示一份后端开发样例
2. 切换到实施交付样例，证明不同输入会得到不同结果
3. 再上传一份本地简历文件，展示系统完整能力
4. 最后打开历史记录和导出按钮，体现交付能力

## 8. 后续可扩展方向

- 增加更多岗位模板
- 做原始 JD 检索与岗位模板映射
- 增加更细的能力图谱与可解释规则
- 增加企业侧岗位定制推荐

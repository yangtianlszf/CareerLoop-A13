# A13 最终答辩演示脚本

## 一、演示前准备

1. 启动本地服务
2. 浏览器打开首页
3. 确认左侧“环境自检”中：
   - 岗位模板库正常
   - 全量 JD 数据集正常
   - 数据库正常
4. 准备好 3 份样例简历

## 二、推荐演示顺序

### 第 1 步：展示系统首页

- 先讲一句项目定位：我们做的是“大学生职业规划智能体”
- 指出系统同时具备：
  - 简历解析
  - 岗位匹配
  - 职业规划
  - 原始 JD 检索
  - 报告导出

### 第 2 步：用后端样例做首次完整演示

- 点击 `后端开发样例`
- 解析模式选择 `auto`
- 点击 `生成职业规划`
- 展示：
  - 主推荐岗位
  - Top 匹配岗位
  - 职业图谱
  - 行动计划
  - 报告导出按钮

### 第 3 步：强调解释性

- 打开 Top1 岗位卡片
- 讲：
  - 共享技能证据
  - 关键能力差距
  - 风险提示
  - 推荐补强项目

### 第 4 步：证明用了官方数据

- 在 `官方 JD 检索` 中输入 `Java`
- 展示检索出的原始 JD
- 点击岗位卡片中的 `查看原始 JD`
- 展示模板证据回看区
- 说明：
  - 我们的岗位模板不是主观设定
  - 是从官方 JD 数据中提炼出来的

### 第 5 步：切换另一个学生样例

- 切换到 `前端产品样例` 或 `实施交付样例`
- 再次生成职业规划
- 说明不同输入会得到不同推荐结果

### 第 6 步：上传本地简历文件

- 演示上传 `docx` 或 `txt`
- 解析完成后直接进入职业规划流程
- 说明系统支持真实场景下的文件输入

### 第 7 步：展示历史记录和导出

- 打开左侧 `最近分析记录`
- 回放上一条分析结果
- 点击导出：
  - HTML
  - Word
  - PDF

## 三、每一段建议怎么讲

- 首页：讲系统能力边界
- 首次分析：讲完整业务闭环
- 匹配解释：讲可解释性
- JD 检索：讲官方数据利用
- 第二个样例：讲泛化能力
- 文件上传：讲落地能力
- 历史与导出：讲交付能力

## 四、现场稳妥策略

- 网络不稳时切换到 `rule`
- 优先使用准备好的样例
- 导出 PDF 时直接使用浏览器打印保存

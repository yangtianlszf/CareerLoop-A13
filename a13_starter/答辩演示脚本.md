# A13 答辩演示脚本

本文件用于现场演示，建议总时长控制在 `6-10` 分钟。核心思路是：先展示主流程，再展示解释性，最后展示交付完整性。

## 1. 演示前准备

开始前先确认：

- 服务已启动
- 浏览器可访问首页
- `http://127.0.0.1:8000/api/system-check` 正常
- 已准备至少 `3` 份样例简历
- 历史记录区里最好已经有 1-2 条分析结果

## 2. 推荐演示顺序

### 第一步：开场

建议开场只讲两句话：

- 我们做的是基于官方 JD 数据的大学生职业规划智能体
- 它不仅给岗位推荐，还能给出解释、行动方案和可导出的职业规划报告

### 第二步：用后端样例展示完整链路

推荐使用：

- `demo_resume_backend.txt`

操作：

1. 点击后端样例
2. 解析模式选择 `auto`
3. 点击 `生成职业规划`

展示重点：

- 主推荐岗位
- 备选岗位
- 主岗适配分
- 推荐摘要

讲解重点：

- 这不是只看关键词，而是基于学生画像和岗位模板做综合判断

### 第三步：展示推荐解释

切到总览中的推荐解释区域，重点讲：

- 为什么主岗排第一
- 为什么备选岗没有排第一
- 共享技能是什么
- 关键差距是什么

如果页面中有雷达图或岗位对照：

- 直接讲主岗和备选岗的能力差异

### 第四步：展示行动路径

切到 `行动路径` 页，重点展示：

- 当前岗位的行动时间线
- 简历改写点
- 面试题板
- 岗位切换模拟器

讲解重点：

- 这个系统不是静态推荐，切换不同岗位时，行动计划会联动变化

### 第五步：展示证据基线

切到 `证据基线` 页，重点展示：

- 岗位基线
- 模板证据
- 原始同名岗位样本
- 本地 JD 检索

讲解重点：

- 推荐结果能够回看官方岗位样本
- 我们能解释“为什么推荐这个岗位”，而不是只给黑箱分数

### 第六步：切换第二个样例

推荐切换：

- `demo_resume_implementation.txt`
  或
- `demo_resume_frontend.txt`

讲解重点：

- 输入不同，主推岗位会发生变化
- 行动方案、证据和作战包会随岗位联动变化

### 第七步：展示图谱与报告

切到 `图谱与报告` 页，重点展示：

- 岗位关系图谱
- 报告预览
- 导出按钮

建议现场演示一个导出动作：

- 导出 PDF
  或
- 导出 Word

### 第八步：展示历史与复核

如果时间足够，再展示：

- 历史记录
- 复核入口
- 留痕结果

讲解重点：

- 这套系统除了学生端，还考虑了学校老师和就业中心的复核场景

## 3. 建议使用的样例顺序

最稳的顺序如下：

1. `demo_resume_backend.txt`
2. `demo_resume_implementation.txt`
3. `demo_resume_frontend.txt`

如果还要补展示：

4. `demo_resume_data_analyst.txt`
5. `demo_resume_testdev.txt`

## 4. 每一步对应的话术重点

- 首页：讲项目定位
- 主流程：讲业务闭环
- 推荐解释：讲可信度
- 行动路径：讲实用性
- 证据基线：讲官方数据利用
- 第二个样例：讲泛化能力
- 导出与复核：讲交付完整性

## 5. 现场兜底策略

### 网络正常

- 使用 `auto`
- 现场展示完整生成流程

### 网络波动

- 切换到 `rule`
- 继续展示样例、证据、行动路径和导出

### 时间不够

- 只演示一个主样例
- 必须保留“推荐解释 + 证据基线 + 导出”三个环节
